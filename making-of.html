03 · Methodology

WHY SCRUM?

Scrum is a framework for delivering complex projects in short, focused cycles called Sprints. We apply it to every client project — and to ourselves. Here's what it means in practice.

📋
PRODUCT BACKLOG

The complete list of everything that needs to be built. Prioritised, estimated, owned. Nothing gets built without being in the backlog. No surprises, no scope creep.

⚡
SPRINT

A fixed time-box — typically 1–2 weeks — in which a defined set of work gets completed. Each sprint delivers something tangible. You don't wait for "done" — you see progress continuously.

🔍
REVIEW & RETRO

After each sprint: what was delivered, what worked, what didn't. Continuous improvement is built into the process. Not a post-mortem — a living habit.

🎯
SPRINT PLANNING

Before each sprint: we define exactly what gets done, who does it, and what "done" means. No ambiguity. No open-ended tasks. Clear scope, clear output.

🔮
TRANSPARENCY

Every task, every decision, every blocker is visible. Clients see the board. Not a curated highlight reel — the actual work, in real time, via Proyex.

⚙️
ITERATIVE DELIVERY

We don't build everything and then show you. We build in layers. Each iteration is functional. Each version is better than the last. v21 is proof.

04 · History

HOW IT HAPPENED.

November 2025
Concept & First Build
Silentmonkey.io v1 launched on Squarespace. Animated timeline, countdown timers, British English copy. Positioned toward artists and coaches. Wrong direction — pivot required.
January 2026
Repositioning
Three-pillar model defined: Consulting · Solutions · Creation. "Eye of the Storm" philosophy developed. Legal contract templates, welcome flows. The brand voice found its register.
February 2026
Ghost Migration Decision
Squarespace replaced by Ghost CMS. Better SEO, built-in newsletter, open architecture. Custom theme development begins. Tool stack locked: Ghost + Proyex.io + Proton Mail.
March 2026
Audit Tool Development
Website audit tool built from scratch — 8 iterations. PageSpeed Insights, Core Web Vitals timeline, hover info panels, trilingual UI. Deployed as lead generation funnel on audit.silentmonkey.io.
April 2026 · Week 1
Style Guide & Strategy
Style Guide v1 completed — four pillars, color system, typography. Two-pillar consulting strategy defined: Audit (leads) → cal.eu → Proyex (delivery). Domain architecture clarified.
April 2026 · Week 2
Homepage v17 — First Deploy
17 iterations in one session. Dot-navigation, onboarding timeline with Scrum badges, two-pillar intake, three contact blocks. First version reviewable on iPad — fast, clean, no agency.
April 2026 · Week 3
Vercel + GitHub · v21 · Two Pillars
Migrated from Netlify to Vercel with GitHub auto-deploy pipeline. Homepage reached v21 — fully English, trilingual language switcher functional. Strategic decision: two pillars (Consulting + Creation), Solution dropped. audit.silentmonkey.io now serves both Consulting landing page and Audit Tool.
